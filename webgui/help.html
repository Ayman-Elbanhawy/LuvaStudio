LUVA Studio
Local offline OT / ICS packet analysis with dashboards, topology views, anomaly review, and HTML reports — all from one interface.

How to use LUVA Studio

  1. Launch the app with StartMe.bat in the LUVA Studio folder.
  2. When the browser opens, go to the Mission Control section and either choose a sample capture or upload your own .pcap, .pcapng, or .gz file.
  3. Click Run Luva to execute the offline analysis pipeline.
  4. Review KPI cards, Overview, Assets, Flows, Topology, Findings, and the embedded Report tab.
  5. Use the built-in Help button inside the GUI anytime you want this guide again.
Tip: LUVA Studio automatically uses an available localhost port now, so startup is more reliable even if the old port is already occupied.

What the tabs are for

  • Overview shows analysis metadata, packet totals, protocols seen, and recent uploads.
  • Assets lists discovered devices, roles, protocols, risk, and open ports.
  • Flows shows communication paths, traffic volume, and protocol usage.
  • Topology embeds the communication map and opens the GraphML output.
  • Findings surfaces audit findings for quick review.
  • Report embeds the latest full HTML report directly inside the dashboard.
Best for demos Start on Overview, then jump to Topology and Report for the most visual walkthrough.
Best for triage Use Findings + Flows together to spot suspicious communication quickly.
Best for validation Use Assets and Report tabs to confirm devices and exported evidence match expectations.

Troubleshooting

  • If the UI does not appear right away, wait a few seconds for the Python environment and local web server to finish starting.
  • If analysis fails, check the log panel inside LUVA Studio and the files under artifacts\logs.
  • If a file does not upload, confirm it exists and uses a supported extension.
  • If you want to share instructions with others, this help page can be printed or saved as PDF from the browser.
